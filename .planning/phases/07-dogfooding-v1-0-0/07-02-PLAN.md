  <action>
Build a local MCPB bundle for installation into Claude Desktop. The release workflow handles this automatically on tag push, but for the dogfood window we build locally.

1. Build the darwin/arm64 binary (assuming macOS development machine):
   ```
   mkdir -p dist/server/darwin/arm64
   CGO_ENABLED=0 GOOS=darwin GOARCH=arm64 go build \
     -ldflags="-s -w -X main.Version=dogfood" \
     -o dist/server/darwin/arm64/version-check-mcp \
     ./cmd/version-check-mcp
   ```
   If on darwin/amd64, use GOARCH=amd64 instead.

2. Create the manifest.json for the MCPB bundle:
   ```
   cat > dist/manifest.json << 'EOF'
   {
     "manifest_version": "0.3",
     "display_name": "version-check-mcp",
     "description": "Gives AI coding agents a single source of truth for software package versions across NPM, PyPI, Go Modules, GitHub Actions, and Maven.",
     "version": "dogfood",
     "server": {
       "type": "stdio",
       "entry_point": {
         "darwin": {
           "arm64": "server/darwin/arm64/version-check-mcp",
           "amd64": "server/darwin/amd64/version-check-mcp"
         }
       }
     }
   }
   EOF
   ```

3. Pack the bundle using @anthropic-ai/mcpb:
   ```
   cd dist && npx @anthropic-ai/mcpb pack . --output version-check-mcp-local.mcpb
   ```
   If npx is not available: `npm install -g @anthropic-ai/mcpb && mcpb pack . --output version-check-mcp-local.mcpb`

4. Verify the bundle was created:
   ```
   ls -lh dist/version-check-mcp-local.mcpb
   ```

**Install the MCPB bundle:**

1. Open Claude Desktop
2. Go to Settings → Extensions (or use "Install Extension" if you have the option)
3. Drag `dist/version-check-mcp-local.mcpb` onto the Claude Desktop window, OR use:
   - macOS: `open dist/version-check-mcp-local.mcpb` (if Claude Desktop is set as the handler)
   - Alternatively: Claude Desktop → Settings → MCP Servers → Add → browse to the .mcpb file

4. **If quarantine error on macOS**, run:
   ```
   xattr -d com.apple.quarantine dist/server/darwin/arm64/version-check-mcp
   ```
   Then retry installation.

**Verify tools appear:**

5. In Claude Desktop, start a new conversation
6. Click the tools/extensions icon (paperclip or tool icon near the input box)
7. Confirm you see BOTH:
   - `validate_version` — with description mentioning "package version", "ecosystem", etc.
   - `get_latest_version` — with description mentioning "latest stable version"

**Verify a tool call works:**

8. Ask Claude: "Using the validate_version tool, check if npm package react version 18.3.1 exists"
9. Expected: Claude calls `validate_version`, returns `{"exists": true, ...}` — NOT an error

**Verify logs go to stderr (not stdout):**

10. In a terminal, run the binary manually and check stderr output:

    ```
    cat stderr.log
    ```
    `stderr.log` should contain JSON slog output. stdout should contain only the JSON-RPC response.

11. Update `docs/DOGFOOD.md` Setup section - check off all three setup items.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Claude Desktop → binary | Claude Desktop spawns the binary as a child process via stdio; the binary receives untrusted tool call inputs |
| local disk → Claude Desktop | MCPB bundle is read from local disk; no network involved in this step |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-03 | Elevation of Privilege | binary spawned by Claude Desktop | accept | Binary has no elevated permissions; runs as user process |
| T-07-04 | Information Disclosure | stderr logs | accept | Logs contain only package names and versions — no credentials, PII, or secrets |
| T-07-05 | Tampering | local .mcpb bundle | accept | Author-only, local build; no supply chain exposure |
</threat_model>
