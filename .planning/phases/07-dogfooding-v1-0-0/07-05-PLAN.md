```markdown
# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.0] - {TODAY_DATE}

### Added
- MCP server over stdio with `validate_version` and `get_latest_version` tools
- NPM registry adapter: validate exact versions, get latest stable/prerelease, major/minor filter, scoped package support
- PyPI adapter: PEP 440 normalization, yanked release detection (PEP 592), `pypi-yanked` source tag
- Go Modules adapter: `v` prefix handling, `+incompatible` preservation, pseudo-version exclusion, capital module escape
- GitHub Actions adapter: tag-based listing with `/releases/latest` hint, rate-limit detection with `X-RateLimit-Reset` surfacing
- Maven Central adapter: `group:artifact` ID format, dot-to-slash path encoding, SNAPSHOT filtering
- LRU+TTL cache with singleflight dedup (success TTL 15 min, miss TTL 2 min, error not cached)
- Structured alternatives on miss: `latest_stable`, `nearest_semver`, `latest_in_major` with 3-tier distance ranking
- Structured error types: `rate_limited`, `not_found`, `upstream_down`, `invalid_input`
- `--cache-ttl` and `--verbose` CLI flags
- GoReleaser multi-arch release pipeline (darwin/amd64, darwin/arm64, linux/amd64, linux/arm64, windows/amd64)
- MCPB bundle (`.mcpb`) for one-click Claude Desktop installation
- SHA256 checksums for all release artifacts

[1.0.0]: https://github.com/JesperRossen/version-check-mcp/releases/tag/v1.0.0
```

Replace `{TODAY_DATE}` with today's date in `YYYY-MM-DD` format.
