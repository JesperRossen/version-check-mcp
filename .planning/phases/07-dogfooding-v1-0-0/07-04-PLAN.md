
   ```
   grep -i "rate.limit\|not.found\|upstream.down\|rate_limited\|not_found\|upstream_down" docs/DOGFOOD.md
   ```
   If no such events were observed naturally, add a note in DOGFOOD.md under the relevant day entry: "Manually triggered: ran `validate_version` with a deliberately wrong version to confirm `not_found` error is observable in stderr logs."

5. If all checks pass, update DOGFOOD.md verdict section:
   - Set "Window complete?" to "yes"
   - Set "P0 bugs open?" to "none" (or list them if any)
   - Set "Ready to tag v1.0.0?" to "yes" (if no open P0s)

If fewer than 7 entries exist: stop here and output a message to the user: "Dogfood window not complete yet. Come back after Day 7 and re-run this plan."
