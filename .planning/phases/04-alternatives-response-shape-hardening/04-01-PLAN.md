  <action>
Create `internal/filter/nearest.go`:

1. Define `AlternativeEntry` struct with `Version string` and `Reason string` json tags.
2. Define reason constants: `ReasonLatestStable = "latest_stable"`, `ReasonNearestSemver = "nearest_semver"`, `ReasonLatestInMajor = "latest_in_major"`.
3. Implement `NearestVersions(versions []string, target string, vPrefixed bool, latestStable string) []AlternativeEntry`:
   - Normalize target with VPrefix if !vPrefixed. If target is not valid semver after normalization, return just `[{latestStable, latest_stable}]`.
   - Filter input: skip invalid semver, skip prereleases, skip pseudo-versions (same logic as FilterAndPickHighest).
   - Build result starting with `{latestStable, "latest_stable"}` (always first, per D-NEAREST-04).
   - Find nearest_semver: sort candidates by distance from target using 3-tier ranking (patch distance in same minor → minor distance in same major → major distance). Prefer higher when tied. Pick the top candidate that isn't already latestStable.
   - Find latest_in_major: highest version in target's own major (per specifics section — most useful for agents doing pinned-major updates). Skip if it equals latestStable or nearest_semver already in the list.
   - Deduplicate: never repeat a version string already in the result slice.
   - Return 1-5 entries (no padding).

Distance calculation helper:
```go
func versionDistance(a, b string) (patchDist, minorDist, majorDist int)
```
Parse major/minor/patch from `semver.Major(v)`, `semver.MajorMinor(v)`, and the remainder. Use `strconv.Atoi` on the numeric parts after stripping the "v" prefix.

Create `internal/filter/nearest_test.go` with table-driven tests covering all behaviors above. Follow exact patterns from `filter_test.go` (package `filter_test`, parallel subtests, `t.Fatalf`/`t.Errorf`).
